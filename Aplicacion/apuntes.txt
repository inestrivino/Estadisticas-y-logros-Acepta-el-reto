* Apuntes sobre la implementacion:

 - vite para contruir el frontend con react

 - rimraf para que se pueda quitar la carpeta dist en windows y linux (dentro del script que construye el proyecto)

 - nodemon para detectar cambios en los archivos y volver a construir con el script
   y luego volver a lanzar el servidor

 - cross-env para poder poner el comando con las variables de entorno en windows y linux
 - dot-env para cargar variables de entorno desde un archivo .env

 - concurrently para lanzar la aplicacion en desarrollo con un solo comando

 - Funcionamiento de como se lanza el codigo en los dockers:
   La aplicacion se lanza corriendo en dockers, uno para el servidor y otro para la base de datos en redis.
   Se lanzan con docker compose configurado en compose.yaml, en el archivo Dockerfile esta la configuracion
   de como se crea la imagen del docker del servidor.
   Esta tiene habilitado el puerto 8080 para los sockets y el 9229 para conectarse con el debugger de VSCode.
   En el launch.json esta especificado como se conecta el debugger.

   Mas en detalle como funciona:
   1. Crea la imagen y la lanza con el código del proyecto de la maquina local como un volumen
   2. Ejecuta "npm run dev" dentro del docker (Lanza el servidor de vite y el back)
   3. El del back lo que hace es pasar el typescript al javascript y lo ejecuta, esto con nodemon
      asi que cada vez que detecta un cambio vuelve a hacer esto
   4. Se conecta el debugger una vez está lanzada la aplicacion con el docker compose,
      esto permite poner punto de ruptura y debuggear normalmente el codigo
   5. Como comparten el codigo con un volumen se detectan los cambios y nodemon vuelve 
      a lanzar la aplicacioon con los cambios

 - recharts para hacer graficos en react

 - redis para la base de datos

 - socket.io para el uso de web-sockets
   - Cada grafico tendria que escuchar un evento y desde el backend se lanza el evento
     correspondiente

  * Antes de cambiar a tsc y tsx

  - comando dev en package.json
    "dev": "cross-env NODE_NO_WARNINGS=1 NODE_ENV=development nodemon --watch"

  - archivo nodemon.json
    {
      "watch": ["src"],
      "ext": "ts",
      "ignore": ["dist", "node_modules"],
      "legacyWatch": true,
      "exec": "tsc && node --inspect=0.0.0.0:9229 dist-server/server.js"
    }
